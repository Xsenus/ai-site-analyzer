# 4.2. Используемые технологии

## Хранилище данных

SQL-хранилище: PostgreSQL 15+ (основная транзакционная БД).

## Векторный индекс

pgvector в составе PostgreSQL, режим эксплуатации — общий кластер БД с
выделенной схемой под векторные таблицы; индексы HNSW/IVFFlat для поиска по
косинусной близости, регламентные ночные пересчёты и бэкапы на уровне СУБД.

## Backend / frontend

Backend: Python 3.11 + FastAPI (async), pydantic settings, Uvicorn/Gunicorn;
в продакшене допускается разделение на сервис интеграций и stateless AI-контур.

Frontend: React (web-кабинет).

## Очереди / планировщик задач

Очередь фоновых задач и пересчётов: Celery + Redis, планировщик — Celery Beat
(с возможностью вынести тяжёлые джобы в отдельные воркеры).

## Интеграции

CRM: Bitrix24 и/или amoCRM (импорт лидов, запись рейтингов/рекомендаций/статусов).

API-обогащение по ИНН (реестровые данные): DaData, Контур.Фокус, СПАРК-Интерфакс
(финальный набор провайдеров выбирается по SLA/стоимости/полноте данных).

LLM API: OpenAI (GPT-4o для извлечения фактов/генерации, text-embedding-3-large
для embeddings) с возможностью подключения внутреннего провайдера.

## Безопасность / аудит

Роли и права доступа, журнал действий, логирование шагов пайплайна, лимиты по
квотам и rate-limit для публичных API.

# 4.3. Публикации/патенты/ПО Заявителя

Патент/заявка: заявка на патент (компьютерно-реализуемый способ/система), готовая
для подачи.

Номер/дата/наименование: заявка к подаче в 2024 г.; номер будет присвоен при
регистрации. Рабочее наименование: «Способ автоматизированной оценки потенциала
B2B‑клиента на основе анализа веб‑источников и реестровых данных».

ПО для ЭВМ: программный комплекс «AI Site Analyzer»; план регистрации ПО — после
стабилизации MVP и внедрения в пилотные интеграции.

Иные результаты ИС: база данных нормализованных карточек компаний, наборы
промптов/шаблонов для LLM, справочники отраслевых классификаторов, эксплуатационная
документация и регламенты качества данных.
